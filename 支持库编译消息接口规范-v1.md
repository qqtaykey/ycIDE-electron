# 支持库编译消息接口规范 v1

## 目标

让第三方支持库按统一协议提供组件事件绑定信息后，IDE 编译器可直接生成事件分发代码，避免每次新增事件都修改编译器硬编码转换层。

## 适用范围

- Windows 窗口程序编译流程
- 支持库窗口组件事件分发
- 第三方 .fne 支持库扩展

## 协议文件位置

编译器会在支持库 .fne 同目录按以下顺序查找协议文件（命中即停止）：

1. `<库文件名>.events.json`
2. `<库文件名>.protocol.json`
3. `<库文件名>.compile-protocol.json`

示例：

- `ycui.fne` 对应 `ycui.events.json`

## JSON 结构

```json
{
  "version": "1.0",
  "eventBindings": [
    {
      "library": "ycui",
      "unit": "ycUI按钮",
      "unitEnglishName": "ycButton",
      "event": "被单击",
      "channel": "WM_COMMAND",
      "code": "BN_CLICKED"
    },
    {
      "library": "ycui",
      "unit": "滑块条",
      "unitEnglishName": "SliderBar",
      "event": "位置被改变",
      "channel": "WM_HSCROLL"
    }
  ]
}
```

## 字段定义

### 顶层字段

- `version`: 可选，协议版本号（字符串或数字）
- `eventBindings`: 必填，事件绑定列表

### eventBindings 条目字段

- `library`: 可选，支持库文件名（不含 `.fne`），建议填写
- `unit`: 必填，组件中文名称（对应支持库窗口组件名）
- `unitEnglishName`: 可选，组件英文名（用于多语言兼容）
- `event`: 必填，事件中文名（与支持库事件定义一致）
- `channel`: 必填，消息分发通道，可选值：
  - `WM_COMMAND`
  - `WM_NOTIFY`
  - `WM_HSCROLL`
  - `WM_VSCROLL`
- `code`: 条件必填
  - 当 `channel` 为 `WM_COMMAND` / `WM_NOTIFY` 时必须提供（如 `BN_CLICKED`、`LVN_ITEMCHANGED`）
  - 当 `channel` 为 `WM_HSCROLL` / `WM_VSCROLL` 时可省略

## 编译器行为

1. 先按协议文件匹配事件绑定。
2. 协议命中时，直接按 `channel + code` 生成事件分发代码。
3. 协议未命中时，回退到旧硬编码规则（兼容老项目）。
4. 若协议和硬编码都无法解析，输出编译告警，提示补充 `eventBindings`。

## 第三方开发要求

1. 组件事件名称需稳定，避免版本间频繁变更。
2. 每个可触发事件都要在协议中给出绑定规则。
3. 发布支持库时，`.fne` 与协议 JSON 必须一起分发。
4. 协议内 `library` 建议与 `.fne` 文件名一致，避免歧义。

## 版本演进建议

- v1: 仅标准消息通道绑定（当前已支持）
- v2: 增加控件类名/样式映射协议（减少 `getWin32ClassName/getWin32Style` 硬编码）
- v3: 增加自定义消息通道（如 `WM_APP + n`）和参数提取规则
